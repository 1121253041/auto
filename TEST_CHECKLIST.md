# 功能测试清单

## 测试环境
- 前端应用: http://localhost:3000
- CORS代理: http://localhost:3001

## 测试步骤

### ✅ 1. 基础功能测试

#### 1.1 应用启动
- [ ] 打开浏览器访问 http://localhost:3000
- [ ] 页面正常加载，显示"Multi-Page Viewer"标题
- [ ] 显示空状态提示（如果没有保存的配置）

#### 1.2 添加网页
- [ ] 点击"添加网页"按钮
- [ ] 输入URL：`https://www.example.com`
- [ ] 输入标题：`示例网站`
- [ ] 点击"添加"按钮
- [ ] 网页卡片出现在界面上
- [ ] iframe开始加载，显示加载动画
- [ ] 加载完成后显示网页内容

#### 1.3 添加第二个网页（测试代理）
- [ ] 再次点击"添加网页"
- [ ] 输入URL：`https://www.baidu.com`
- [ ] 输入标题：`百度`
- [ ] 勾选"使用代理"
- [ ] 点击"添加"
- [ ] 观察是否正常加载（可能需要在代理中查看日志）

#### 1.4 响应式布局测试
- [ ] 添加第3个网页，观察布局变为2列
- [ ] 添加第4个网页，观察布局保持2列
- [ ] 添加第5个网页，观察布局变为3列
- [ ] 删除部分网页，观察布局自适应变化

#### 1.5 全屏功能
- [ ] 点击任意网页卡片的全屏按钮（⤢）
- [ ] 该iframe全屏显示，其他iframe隐藏
- [ ] 再次点击全屏按钮（⤓）
- [ ] 退出全屏，恢复网格布局

#### 1.6 删除网页
- [ ] 点击任意网页卡片的删除按钮（✕）
- [ ] 该网页从列表中移除
- [ ] 布局自动调整

#### 1.7 编辑网页
- [ ] 点击"配置"按钮
- [ ] 在配置面板中修改某个网页的URL或标题
- [ ] 点击"更新"
- [ ] 观察iframe重新加载新URL

#### 1.8 持久化存储
- [ ] 添加几个网页
- [ ] 刷新页面（F5）
- [ ] 验证所有网页配置仍然存在

### ✅ 2. 代理功能测试

#### 2.1 全局代理开关
- [ ] 打开配置面板
- [ ] 切换"全局使用代理"开关
- [ ] 添加新网页
- [ ] 观察新网页默认是否使用代理设置

#### 2.2 单个网页代理
- [ ] 添加一个网页时勾选"使用代理"
- [ ] 观察iframe是否显示代理图标（🔌）
- [ ] 检查代理服务器日志是否有请求记录

#### 2.3 代理服务器测试
- [ ] 打开新的命令行窗口
- [ ] 查看代理服务器输出日志
- [ ] 验证有代理请求记录

### ✅ 3. 插件系统测试

#### 3.1 插件管理面板
- [ ] 点击右上角"🔌 插件"按钮
- [ ] 插件管理面板弹出
- [ ] 显示已安装的插件列表

#### 3.2 示例插件验证
- [ ] 查看是否显示"Web Monitor"插件
- [ ] 显示插件版本号（v1.0.0）
- [ ] 显示插件描述

#### 3.3 插件功能验证
- [ ] 添加一个新网页
- [ ] 打开浏览器开发者工具（F12）
- [ ] 查看Console控制台
- [ ] 验证是否有"[Web Monitor]"相关日志
- [ ] 验证是否有"Web Monitor Plugin 已加载"提示

#### 3.4 插件卸载（可选）
- [ ] 在插件管理面板中点击"卸载"按钮
- [ ] 确认卸载对话框
- [ ] 插件从列表中移除
- [ ] 刷新页面后插件不再工作

### ✅ 4. 错误处理测试

#### 4.1 无效URL

- [ ] 观察是否显示加载失败提示
- [ ] 验证错误状态UI

#### 4.3 空状态
- [ ] 删除所有网页
- [ ] 验证显示空状态提示
- [ ] 提示用户添加第一个网页

### ✅ 5. 性能测试

#### 5.1 多网页加载
- [ ] 添加5-6个网页

- [ ] 验证是否有明显卡顿

#### 5.2 切换全屏
- [ ] 在多个iframe间快速切换全屏
- [ ] 验证响应速度
